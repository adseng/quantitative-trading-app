# 多因子量化分析工具 - 规划文档

## 一、核心目标

做一个**多因子量化分析工具**：  
把各类技术指标因子转化为**看涨/看跌**两个维度，用加权分数预测下一根 K 线方向。

---

## 二、逻辑框架

```
输入: 各类技术指标因子
  ↓
每个因子产出信号: 0（横盘）、1（看涨）、-1（看跌）
  ↓
影响力 = 权重 × 信号
  ↓
看涨/看跌总积分 = 各因子影响力之和
  ↓
信号: 当前 K 线收盘时
  - 看涨总积分 > 看跌总积分 → 预测下一根 K 线看涨
  - 反之 → 预测下一根 K 线看跌
  ↓
验证: 回测 + 自动调参
```

---

## 三、可调节参数

| 参数 | 说明 | 示例 |
|------|------|------|
| K 线周期 | 预测的时间维度 | 15 分钟（可改为 5m、1h 等） |
| 因子权重 | 每个因子对涨跌的贡献 | 可调，通过回测自动优化 |

## 七、关键点小结

1. **因子库**：需要系统收集和筛选各类技术指标因子，并设计可扩展的因子接口。  
2. **权重学习**：权重通过回测迭代调整，而不是主观定死。

4. **测试网先行**：所有合约数据先从测试网获取，降低成本和风险。

```go
// 信号评分上下文
type SignalContext struct {
    BullScore float64
    BearScore float64

    UseBreakout bool; BreakoutPeriod int; BreakoutWeight float64
    UsePriceVsMA bool; PriceVsMAPeriod int; PriceVsMAWeight float64
    UseATR bool;    ATRPeriod int;        ATRWeight float64
    UseVolume bool; VolumePeriod int;     VolumeWeight float64
    UseSession bool; SessionWeight float64
}
```

### 因子信号模型

每个因子输出三值信号：**0（横盘）、1（看涨）、-1（看跌）**。

- **影响力** = 权重 × 信号
- **信号分数** = 各因子看涨影响力之和 - 看跌影响力之和

### 流水线调用

```go
// 链式调用
ctx.FactorMA(5, 20, 1.0).FactorRSI(14, 70, 30, 1.0).FactorMACD(12, 26, 9, 1.0)...

// 或用统一配置
EvaluateWithConfig(kl, &FactorConfig{...})
```

### 核心发现：BTC 15m 是短线均值回归市场

所有趋势跟随因子在教科书方向上的信号正确率均 **低于 50%**，而反向（负权重）后均 **高于 50%**。

### 关键洞察

1. **信号正确率 vs 信号率的权衡**：Boll P15 有 58.8% 正确率但只有 3.4% 信号率（每30根K线才出1次信号）。RSI 60/40 正确率降到 53.3% 但信号率提升到 52.5%。实际交易需要在两者间找平衡。

2. **"总正确率"是误导指标**：当信号率很低时，`correct/total` 会极低（RSI 看起来只有10%），但实际上在发出信号的时候有 54% 正确率。因此移除总正确率，只保留信号正确率。

3. **负权重不是"反向操作"**：最初想用 `Reverse` 布尔标记翻转预测结果，但这是粗暴的。正确做法是让每个因子的权重可以为负值（`weight * signal`），这样可以精细控制每个因子的贡献方向。

4. **多因子组合时信号率会变化**：多个因子叠加后，BullScore 和 BearScore 同时为0的概率降低，信号率会上升。但正确率可能被弱因子稀释。

### 新增指标说明

| 指标 | 含义 | 用途 |
|------|------|------|
| 信号正确率 | `correct / signalCount` | 核心指标，只算有信号时是否预测对 |
| 有效预测数 | `signalCount` | 非零预测的总次数 |
| 平均净分 | `avg(BullScore - BearScore)` | 正=整体偏看涨，负=偏看跌 |
| 信号强度 | `avg(|BullScore - BearScore|)` | 值越大=模型越"有把握" |


2. **权重幅度不影响单因子正确率**：Break10 w=-1 和 w=-2 都是 57.5%。因为单因子下只有 +1/-1/0 信号，权重只放大分数绝对值，不改变方向。**权重仅在多因子组合中影响最终投票**。

3. **Boll + Breakout(-) 是最强二因子组合**：57.2% @ 18.5%。两者互补——Boll 捕捉布林带反弹（均值回归），Breakout(-) 捕捉假突破反转。

5. **ATR(-) 是进一步提升信号率的第四因子**：54.2% @ 54.1%。但每加一个因子约掉 1% 正确率。

6. **多因子组合的边际效应递减**：
   - 1因子: 58.1% @ 10.9%
   - 2因子: 57.2% @ 18.5% （-0.9%正确率, +7.6%信号率）
   - 3因子: 55.5% @ 32.9% （-1.7%, +14.4%）
   - 4因子: 54.2% @ 54.1% （-1.3%, +21.2%）

7. **平均净分全部为负**（-0.003 ~ -0.35）：说明整体偏空（BTC 在这段历史数据中偏下行），但绝对值很小，市场并非单边。

## 十四、进化式自动调参（Evolutionary Auto Test）

### 方法论

V1-V4 都是**人工设计用例 → 测试 → 分析 → 人工调整**的手动循环。
从 V5 开始，使用**进化算法**自动完成"分析→调参"步骤，实现自动化参数优化。

### 算法流程

```
Iteration 1: 初始种群 = V4 的 200 组用例
     ↓
  运行 200 组回测 → 得到每组的信号正确率
     ↓
  按信号正确率排序，分析 top 策略
     ↓
  生成下一代种群（200 组）:
    ├── 精英保留 (20):  直接保留 top 20
    ├── 变异 (120):     从 top 30 中变异参数
    ├── 交叉 (40):      从 top 20 中随机配对交叉
    └── 随机探索 (20):  在参数空间随机生成
     ↓
Iteration 2: 用新种群重复上述步骤
     ↓
  ... 循环 N 次 ...
     ↓
  输出趋势表 + 全局最佳策略
```

### 变异策略

| 操作 | 概率 | 幅度 |
|------|:----:|------|
| Boll 周期 ±1~2 | 40% | 整数微调 |
| Boll 乘数 ±0.2 | 40% | 浮点微调 |
| Breakout 周期 ±1~3 | 40% | 整数微调 |
| RSI 周期 ±1~2 | 40% | 整数微调 |
| RSI 阈值 ±3 | 25% | 浮点微调 |
| 权重 ×0.8~1.2 | 20% | 比例缩放 |
| 开关因子 | 8% | 随机添加/移除一个弱因子 |

### 交叉策略

从两个父代中，每个因子模块（Boll/Break/RSI/ATR/...）各有 50% 概率继承自父代1或父代2。
