我已经拿到当前保留下来的核心文件清单，下面直接按目录解释每个文件在新版本里的作用。当前项目已经被收缩成一条很清晰的主链路：

`Binance 拉取 K 线 -> 保存本地文件 -> 策略识别 -> 订单级回测 -> Wails 前端展示`

## 根目录
- `main.go`
  Wails 桌面应用入口。负责启动应用窗口、加载前端静态资源，并绑定 `App` 给前端调用。
- `app.go`
  Wails 暴露层。把前端能调用的方法集中暴露出来，目前主要是：
  - 获取远程 K 线
  - 加载本地 K 线
  - 运行回测
- `go.mod`
  Go 模块定义和依赖声明。
- `README.md`
  当前重构后的项目说明，介绍新的使用方式和命令。

## `cmd`
这是命令行入口层。

- `cmd/fetchdata/main.go`
  拉取 Binance 历史 K 线数据。
  支持配置交易对、周期、数量、输出路径、请求间隔、是否覆盖。
  默认输出到 `docs/db/data-k-{interval}-{count}.txt`。
- `cmd/backtest/main.go`
  回测命令入口。
  从本地 K 线文件读取数据，带上策略参数执行回测，并把结果写到 `docs/db/test-{策略名}.txt`。

## `internal`
这是后端核心逻辑。

### `internal/appservice`
- `internal/appservice/service.go`
  应用服务层。
  作用是把更底层的模块串起来，给 `app.go` 提供统一调用入口。
  目前负责：
  - 调 Binance 拉远程 K 线
  - 调本地文件加载 K 线
  - 调回测引擎执行策略回测

### `internal/binance`
- `internal/binance/client.go`
  Binance 数据访问层。
  负责：
  - 初始化 Binance REST 客户端
  - 读取代理和超时配置
  - 按周期/数量分页抓取 K 线
  - 把 Binance 返回的数据转成项目内部的 K 线结构

### `internal/config`
- `internal/config/config.go`
  配置读取层。
  负责从 `.env.local` / `.env` 读取配置，并提供统一的环境变量 key 常量。

### `internal/datafile`
- `internal/datafile/datafile.go`
  本地数据文件读写层。
  负责：
  - 从 JSONL 文件中读取 K 线
  - 把抓到的 K 线写入本地文件
  - 生成默认数据文件路径

### `internal/market`
- `internal/market/market.go`
  市场基础模型。
  目前定义了 `KLine`，以及一些 K 线基础方法，比如：
  - 判断阳线/阴线
  - 实体大小
  - 振幅
  - 上下影线

这是新项目里最基础的领域模型层。

### `internal/strategy`
- `internal/strategy/types.go`
  策略通用类型定义。
  包括：
  - `Direction`
  - 策略参数 `BoxPullbackParams`
  - 信号结构 `Signal`
  - 默认参数和参数归一化逻辑
- `internal/strategy/box_pullback.go`
  首个策略实现文件。
  实现的是“箱体突破回踩确认”策略，包含：
  - K1 强度过滤
  - 多头/空头信号识别
  - 回踩容差
  - 冷却时间
  - 盈亏比相关信号字段
- `internal/strategy/box_pullback_test.go`
  策略测试。
  用小样本 K 线验证是否能正确识别出做多信号。

### `internal/backtest`
- `internal/backtest/types.go`
  回测层的数据结构定义。
  包括：
  - 回测请求 `RunRequest`
  - 订单记录 `Trade`
  - 汇总统计 `Summary`
  - 完整回测结果 `Report`
- `internal/backtest/engine.go`
  回测引擎核心。
  负责：
  - 从数据文件加载 K 线
  - 调策略生成信号
  - 根据下一根 K 线开盘价入场
  - 按止损/止盈规则模拟平仓
  - 统计胜率、PnL、回撤、余额等
  - 导出结果文件
- `internal/backtest/engine_test.go`
  回测测试。
  用小样本验证是否能正常生成订单并命中止盈。

## `frontend/src`
这是前端源码，已经被收缩成单页工作台。

### 入口与全局
- `frontend/src/main.tsx`
  React 前端入口，挂载应用、配置 Ant Design 和错误边界。
- `frontend/src/style.css`
  全局样式。
- `frontend/src/vite-env.d.ts`
  Vite 类型声明。

### 路由
- `frontend/src/router/index.tsx`
  路由入口。现在只保留首页工作台。
- `frontend/src/router/settings.ts`
  路由常量定义。

### 布局
- `frontend/src/structure/Layout.tsx`
  页面整体框架，顶部菜单 + 主内容区域。
- `frontend/src/structure/Menus.tsx`
  顶部菜单定义。现在只保留“交易工作台”。
- `frontend/src/structure/HeaderMenu.tsx`
  头部右侧占位，目前基本为空。

### Wails 前端桥接
- `frontend/src/api/wails.ts`
  前端调用 Wails 后端的封装层。
  把 `window.go.main.App` 里的方法包装成：
  - `fetchKlines`
  - `loadLocalKlines`
  - `runBacktest`

### 工作台页面
- `frontend/src/views/cozePredict/CozePredict.tsx`
  当前主页面，虽然目录名还叫 `cozePredict`，但内容已经变成量化工作台。
  负责：
  - 输入本地数据路径
  - 拉取远程 K 线
  - 配置策略参数
  - 运行回测
  - 展示 K 线图、统计摘要、订单结果
- `frontend/src/views/cozePredict/types.ts`
  前端类型定义。
  包括：
  - `KLine`
  - 策略参数
  - 信号
  - 回测订单
  - 回测报告
- `frontend/src/views/cozePredict/hooks/useCozeKlineStream.ts`
  K 线数据加载 hook。
  现在主要负责：
  - 从 Binance 拉 K 线
  - 从本地文件读 K 线
  - 管理加载状态
- `frontend/src/views/cozePredict/hooks/useBacktestRunner.ts`
  回测执行 hook。
  负责调用后端回测接口并管理运行状态和错误信息。

### 组件
- `frontend/src/views/cozePredict/components/KlineChart.tsx`
  复用保留下来的 K 线图组件。
  现在除了画蜡烛图和均线，还支持把：
  - 策略信号
  - 开仓点
  - 平仓点
  叠加到图表上。
- `frontend/src/views/cozePredict/components/PredictionResults.tsx`
  名字还没改，但实际展示的是订单结果表。
  显示每笔交易的方向、开平仓时间、价格、止损止盈、盈亏、余额等。

## `frontend/wailsjs`
这是 Wails 自动生成的前端绑定代码，一般不手写。

- `frontend/wailsjs/go/main/App.js`
- `frontend/wailsjs/go/main/App.d.ts`
  前端可直接调用的后端方法声明与桥接代码。
- `frontend/wailsjs/go/models.ts`
  后端结构体对应的前端模型定义。
- `frontend/wailsjs/runtime/runtime.js`
- `frontend/wailsjs/runtime/runtime.d.ts`
  Wails 运行时能力，比如事件、窗口交互等。

## 一句话理解分层
- `cmd/*`：命令行入口
- `app.go`：桌面端对前端暴露的 API
- `internal/binance`：拉行情
- `internal/datafile`：读写本地数据
- `internal/strategy`：产生交易信号
- `internal/backtest`：把信号变成订单和统计
- `frontend/src/views/cozePredict`：把结果展示出来
